# CLAUDE.md — context for AI assistants on this project

> Read this first if you are an AI agent (Claude / Sonnet / etc.) being
> asked to work on this codebase. It explains what we built, how it's
> organized, and the conventions to follow.

## What this is

A modern web reissue of **Guit95** — Ubi Soft's 1995 multimedia CD
"Guitar — discover the songs". Original CD had 7 lessons (Hey Joe,
No Woman No Cry, Life by the Drop, Sweet Home Alabama, Dust in the
Wind, Blowin' in the Wind, Wild World), each with a video lesson,
synchronized tablature, chord database with samples, and per-song
exercises with voice commentary.

This rewrite is **for personal practice only** by the project owner
(who owns the original CD). The original AVI/BMP/WAV assets are NOT
distributed in this repository — only the engine code that knows how
to interpret them. To use the app you need a copy of the original
Guitar.iso (or the physical CD).

## Stack (locked — do not change)

- **Vite + TypeScript** — build & dev server
- **Vanilla DOM** — no React/Vue/Svelte. The app is ~10 screens of
  pixel-perfect 1995 layout; framework gives no benefit.
- **CSS** — handwritten, no Tailwind/SCSS/PostCSS. Variables-based
  theming.
- **Web Audio API** — for chord samples and (deferred) tuner /
  metronome.
- **HTML5 `<video>`** — with `playbackRate` + `preservesPitch` for
  slow-down practice.
- **localStorage** — for settings, progress, last positions.
- **ffmpeg** + **sips** — one-time asset conversion (CD AVI/BMP →
  modern MP4/PNG).
- **No backend, no database, no auth** — fully static SPA.

## Development conventions

### Commit messages

Format: `phase <N>[<letter>]: <one-line summary>`

Examples (real commits in history):
- `phase 0: project skeleton + asset pipeline`
- `phase 2c: page-flip tab scroll + video controls fix`
- `phase 4d: renumber exercises by song-curriculum order`

If the change is not part of a numbered phase, use a plain imperative
("add ...", "fix ...").

### Sonnet sub-agents

Most implementation was done by Sonnet sub-agents reading
self-contained spec files in `phases/`. Pattern:
1. Opus writes `phases/N-spec.md` with: context, goal, scope IN/OUT,
   detailed implementation, DoD checklist, mandatory demo block.
2. Opus spawns a `general-purpose` subagent with `model: "sonnet"`,
   passes path to spec file.
3. Sonnet reads spec + relevant prior phase specs + relevant source
   files, implements, commits, returns demo block + report.
4. Opus verifies (smoke-tests + reads diff), runs the demo, hands to user.

If you continue this project as a Sonnet sub-agent, follow the spec
file given to you exactly. Don't add features not in the spec.

### Per-phase commits

Each spec lists its commit message. Sonnet commits when done. If a
spec covers multiple sub-tasks (Phase 8 has 5 features), commit each
separately.

### Asset pipeline reentrancy

`scripts/build-assets.sh` is idempotent — re-running skips files that
already have a fresher destination. Optional positional args limit
which song slugs to convert.

### Modal/overlay convention

Help, Settings, etc. use `position: fixed; inset: 0; z-index: 1000`
with a centred panel. Esc closes; click backdrop closes.

### Hotkey listening

`window.addEventListener("keydown", handleKey)` — single dispatcher.
Bail out if focus is on `<input>/<textarea>/<select>` or
`contenteditable`. Use `event.code` (layout-independent), not
`event.key`. `e.preventDefault()` for handled keys to suppress browser
defaults.

### Responsive media

Never use fixed `width="640"` on `<video>` or canvas. Use CSS
`width: 100%; height: auto; max-height: ...vh`. For canvas, internal
buffer = source × DPR for sharpness, CSS sizing fluid via
`aspect-ratio`.

## Key learnings during build

- **CD's "exercise N" is by file id, not by song order**. Phase 4d
  added renumber-by-song-order so the first hotspot in the song is
  "Exercise 1" regardless of CD's `exercice=15` (intro).
- **CSS `transition` fights rapid wheel events**. Phase 8 fix:
  toggle `.dragging` class during wheel gesture (debounced re-enable).
- **Browser `controls` attr on `<video>` is invisible if pane height
  clips the video.** Phase 2c fix: `align-items: stretch` +
  `height: 100%` + `object-fit: contain`, drop `overflow: hidden`.
- **macOS Safari needs `webkitPreservesPitch` AND `preservesPitch`**
  for slow-down without chipmunk effect.
- **YIN > FFT for guitar pitch detection** — low E (82 Hz) is below
  FFT bin resolution at typical buffer sizes. (Tuner deferred but
  this is the intended approach.)
- **Pointer Events unify mouse + touch** — cleanest path to iPad
  support; no separate touch handler code.
- **Always commit per feature**, even small ones. We had a Sonnet
  agent time out at ~108min in Phase 7; thanks to incremental
  commits we lost zero code.
