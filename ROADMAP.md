# Guit95 Modern — Roadmap

> Поэтапный план реализации. Каждый этап — самостоятельный вертикальный
> срез: после каждого фазы можно сесть с гитарой и пользоваться. Никакой
> «инфраструктура → потом фичи».
>
> **Цель проекта:** учиться играть на гитаре по урокам с CD 1995 года.
> **Стек:** современный, но без излишеств. Функция важнее технологии.
> **Пользователь:** один человек (вы). Никакой шаринг, аутентификация,
> аналитика.

---

## Стек (зафиксирован)

| Слой           | Выбор                          | Почему                              |
| -------------- | ------------------------------ | ----------------------------------- |
| Билд           | **Vite** + TypeScript          | Hot-reload, минимальный конфиг      |
| UI             | **Vanilla DOM** (без фреймворков) | ~10 экранов, нет смысла в React/Vue |
| CSS            | Plain CSS + CSS variables      | Без Tailwind/SCSS                   |
| Видео          | HTML5 `<video>` + `playbackRate` | Slow-down out-of-the-box            |
| Аудио          | Web Audio API                  | FFT-тюнер, точный метроном          |
| Стейт          | Plain JS объекты + localStorage | Без Redux/Zustand                   |
| Конверсия     | `ffmpeg` (один раз)            | AVI → MP4 (H.264 + AAC)             |
| Деплой         | `vite build` → static          | Можно открывать `file://`, но локалхост лучше для микрофона |
| Тесты          | Минимально, только парсеры     | Это не enterprise — ручной QA на каждом этапе |

**Принципы:**
1. Каждая фаза = ≤8 часов работы Sonnet-агента.
2. После каждой фазы — деплой и реальное использование.
3. Если упёрлись в архитектурный вопрос — спрашиваем пользователя, не додумываем.

## Phase 0 — Asset Pipeline & Project Skeleton

**Цель:** проект компилируется, dev-сервер запускается, тест-страница играет конвертированное видео Hey Joe.

**Включено:**
- Копирование `/Volumes/Guitar/GUITAR/` и `/Volumes/Guitar/VIDEO/` в `~/src/guitar/assets/raw/` (источник истины, не трогаем).
- Скрипт `scripts/build-assets.sh`:
  - `ffmpeg` AVI → MP4 (H.264 + AAC) для 8 main-videos и видео упражнений Hey Joe.
  - Имена в lowercase (для безопасности на case-sensitive ФС).
  - Вывод в `public/assets/`.
- Vite-проект на TypeScript (минимальный `vite.config.ts`, `tsconfig.json`, `index.html`, `src/main.ts`).
- Тест-страница: `<video src="/assets/heyjoe/hjoe.mp4" controls>`.
- `.gitignore` (assets/raw — большой, исключить из VCS).

**Исключено:**
- Любой UI кроме тест-плеера.
- Конверсия BMP (отложено — Safari/Chrome BMP читают, можно отложить до проблемы).
- Tooling вроде ESLint/Prettier (можно добавить позже, сейчас отвлекает).

**Definition of Done (инженерное):**
- [ ] `~/src/guitar/package.json` существует, `npm install` отрабатывает без ошибок.
- [ ] `npm run dev` стартует Vite на localhost:5173.
- [ ] `scripts/build-assets.sh` отрабатывает идемпотентно: повторный запуск не делает лишней работы.
- [ ] `public/assets/heyjoe/hjoe.mp4` существует, размер <50% оригинального AVI.
- [ ] `git status` показывает: `assets/raw/` игнорируется, билд-артефакты игнорируются.

**Acceptance Criteria (пользовательское):**
- [ ] Открываю `localhost:5173` → вижу тест-плеер, нажимаю play → Hey Joe играет со звуком.
- [ ] Перематываю до конца → видео не падает, не зависает.
- [ ] В Safari и Chrome работает одинаково.

**Риски / нюансы:**
- AVI могут содержать audio в кодеке, который ffmpeg по умолчанию не возьмёт — проверить флаг `-c:a aac -b:a 128k`.
- Кодеки Cinepak/Indeo: требуется `ffmpeg` с поддержкой; на macOS из brew по умолчанию работает.

## Phase 1 — Hey Joe MVP

**Цель:** одна страница с Hey Joe видео + 11 аккордов. Клик по аккорду показывает диаграмму и проигрывает сэмпл.

**Включено:**
- Парсер INI (Windows-1252, CRLF, секции, `key=value`).
- Парсер `.CHD` (на основе INI-парсера).
- Layout страницы:
  - Верх: видео с native controls.
  - Низ: ряд из 11 аккордовых кнопок (имена из CHD).
  - Справа от ряда: панель с активной диаграммой аккорда + кнопка «Vert/Horiz».
- Кнопка аккорда → крупная диаграмма + воспроизведение `sound=hj_*.wav` через `Audio` или `<audio>`.
- Тогл «вертикальный/горизонтальный» переключает `pic_rect` ↔ `pic_rect_us`.
- Диаграммы — нарезка из общего BMP `heyjoe2.bmp` (рисуем `<canvas>` с `drawImage(sx,sy,sw,sh)`).

**Исключено:**
- Скролл табулатуры (Phase 2).
- Slow-down (Phase 3).
- Упражнения (Phase 4).

**Definition of Done:**
- [ ] `src/parsers/ini.ts` парсит INI-файлы, юнит-тесты на 3 примерах из репо (1 EXR, 1 CHD, 1 SCO).
- [ ] `src/parsers/chd.ts` возвращает массив `Chord` с типами (id, name, picRect, picRectUS, sound, hand, avi, comments).
- [ ] Главный экран рендерится, видео воспроизводится через `<video>`.
- [ ] 11 кнопок аккордов кликабельны, каждая показывает свою диаграмму.
- [ ] Воспроизведение сэмпла идёт через единственный `AudioContext` (нет наслоения).
- [ ] Переключатель Vert/Horiz работает.
- [ ] Нет ошибок в консоли браузера.

**Acceptance Criteria:**
- [ ] Открываю приложение → вижу видео Hey Joe и ряд кнопок: C, Go, D, A, E, C_3, G_3, D_5, A_5, E_7, E7#9.
- [ ] Жму Play → видео играет с оригинальным саундтреком.
- [ ] Кликаю «C» → справа появляется диаграмма аккорда C, в наушниках слышу сэмпл.
- [ ] Кликаю «Vert» → диаграммы перерисовываются в горизонтальном виде.
- [ ] Кликаю аккорд во время игры видео — сэмпл играет поверх.

**Риски / нюансы:**
- Координаты `pic_rect` и `pic_rect_us` — относительно общего BMP. Проверить, что нарезка попадает в правильный аккорд.
- Сэмплы WAV из CHORDS — некоторые могут быть не Mono/PCM, тогда не сыграют. Перепрогнать через `ffmpeg -ac 1 -ar 44100`.

## Phase 2 — Синхронная скроллящаяся табулатура

**Цель:** под видео скроллится длинная BMP-табулатура. Курсор в центре, картинка движется. Bar-маркеры кликабельны для прыжка.

**Включено:**
- Парсер `.SCO`: header + `[bar]` (массив pixels) + `[event]` (массив `{frame, pixel, chord?, manual?}`) + `[difficulty]` (отложить визуально, но парсить).
- Компонент TabScroller: `<canvas>` или `<div style="overflow:hidden">` с `transform: translateX(-N)`.
- Long-strip изображение `HEYJ-B2.BMP` грузится один раз (или конвертится в PNG для надёжности).
- Алгоритм sync: на `<video>.timeupdate` (или `requestAnimationFrame`) — текущий frame → бинарный поиск в `[event]` → линейная интерполяция → целевой pixel → `translateX`.
- Bar markers: вертикальные тонкие линии поверх табулатуры в местах из `[bar]`.
- Клик по bar marker → `video.currentTime = frameToTime(bar.pixel → ближайший event.frame / fps)`.
- Подсветка активного аккорда: при пересечении event.chord → выделяем кнопку аккорда из Phase 1.
- Получение FPS видео: либо из метаданных AVI (хранить в JSON), либо парсить из `endingframe / durationSeconds`.

**Исключено:**
- Difficulty-hotspots визуально (Phase 4).
- Loop A/B (Phase 3).

**Definition of Done:**
- [ ] `src/parsers/sco.ts` возвращает `Score { header, bars, events, difficulties }`.
- [ ] FPS вычисляется автоматически: `endingframe / videoDuration` (после загрузки `<video>` метаданных).
- [ ] TabScroller компонент: при `currentTime=0` курсор на `startingPixel`, при `currentTime=duration` — на `endingPixel`.
- [ ] Skipping: `requestAnimationFrame` не реже 30 FPS, без подвисаний.
- [ ] Bar markers рендерятся в правильных пиксельных позициях.
- [ ] Клик по bar → видео прыгает, табулатура мгновенно скроллится в нужное место.
- [ ] Активный аккорд подсвечивается (border/glow) на ряду из Phase 1.

**Acceptance Criteria:**
- [ ] Жму Play → видео играет, табулатура движется справа налево синхронно.
- [ ] На паузе табы не двигаются. Перематываю видео ползунком — табы перепрыгивают.
- [ ] Кликаю по bar #5 → видео прыгает в начало 5-го такта, табы скроллятся туда же.
- [ ] Когда играется аккорд C — на нижнем ряду горит подсветка кнопки C; при смене на G — подсветка переезжает на G.
- [ ] Скролл выглядит плавно (нет рывков на >60 Гц мониторе).

**Риски / нюансы:**
- Длина табулатуры ~14000 px — большой `<img>`. Проверить, что Safari/Chrome не упираются. Альтернатива: разрезать на тайлы по 2000px.
- BMP с палитрой 8-bit — Safari иногда показывает с артефактами. Лучше сразу сконвертить в PNG в Phase 0.

## Phase 3 — Slow-down + A↔B Loop (KILLER FEATURE)

**Цель:** учить трудные места на 0.5× в бесконечном цикле между двумя bar-маркерами.

**Включено:**
- Слайдер темпа: 0.25× → 1.5× с шагом 0.05.
- `<video>.playbackRate = N; <video>.preservesPitch = true` (Safari это не уважает по умолчанию — fallback на `webkitPreservesPitch`).
- A↔B Loop UI:
  - Две кнопки «A» и «B».
  - Клик «A» — устанавливает A в текущий bar-маркер (округление).
  - Клик «B» — то же для B.
  - Кнопка-тогл «Loop on/off».
  - Визуально: на табулатуре A и B показаны цветными вертикальными линиями + затенение области между ними.
- Логика loop: на `timeupdate` если `currentTime >= timeOf(B)` → `currentTime = timeOf(A)`.
- «Сбросить A/B» — двойной клик или отдельная кнопка.
- Темп сохраняется в localStorage; loop A/B — нет (это сценарный, не персональный).

**Исключено:**
- Сохранение нескольких именованных loop-ов («моих трудных мест»).

**Definition of Done:**
- [ ] Слайдер темпа меняет `playbackRate` мгновенно.
- [ ] Питч сохраняется на 0.5× (проверить ушами — не падает на октаву).
- [ ] A и B-маркеры рендерятся, перетаскиваются (или клик-устанавливаются).
- [ ] Loop включён → видео гарантированно зацикливается без gap >100мс.
- [ ] Темп сохраняется при перезагрузке страницы.

**Acceptance Criteria:**
- [ ] Ставлю темп 0.5× → видео играет в 2 раза медленнее, голос не превращается в низкое мычание.
- [ ] Кликаю «A» на bar 3, «B» на bar 5, включаю Loop → плеер бесконечно крутит интервал 3–5.
- [ ] Меняю темп на 0.7× — продолжает крутить, но быстрее.
- [ ] Выключаю Loop → плеер продолжает играть с позиции, не прыгает.
- [ ] Перезагрузка страницы → темп остался 0.7× (или последний выбранный).

**Риски / нюансы:**
- Safari `preservesPitch` через `webkitPreservesPitch`. Тестировать на macOS Safari явно.
- На очень низких темпах (<0.4×) браузеры иногда выдают артефакты — задокументировать как known limitation.

## Phase 4 — Exercises + Difficulty Hotspots

**Цель:** все 16 упражнений Hey Joe доступны. Трудные места в песне — кликабельные, ведут в нужное упражнение.

**Включено:**
- Парсер `.EXR` со списком упражнений (на основе INI).
- Минимальный sequence-runner: только опкоды, нужные для упражнений Hey Joe:
  - `aviopen`, `aviplay`, `playsnd`, `stopall`
  - `setbtnseq`, `setmciseq`, `gotoseq`
  - `showpicture`, `hideallpictures`
  - `waitnotify`
- Боковая панель «Упражнения» — список 16 пунктов с озвученными названиями (`%sex1-tit.wav`).
- Клик по упражнению → открывает экран упражнения:
  - Видео упражнения (например `1-1.avi`).
  - Голосовой комментарий в начале.
  - Под видео — статичная картинка табулатуры (`SCO1-2.BMP`).
  - Кнопки «◀ предыдущее / следующее ▶».
  - Возврат «← к песне».
- Difficulty hotspots на основной табулатуре:
  - Полупрозрачные цветные прямоугольники по `[difficulty].rect` поверх tab strip.
  - Hover → показывает «Упражнение N: <название>».
  - Клик → озвучивает голосовую подсказку и открывает упражнение.
- Применяем slow-down/loop из Phase 3 и в режиме упражнения тоже.

**Исключено:**
- Полная sequence-runner для всех 15 опкодов (только подмножество для упражнений).
- Прогресс «упражнение пройдено» (Phase 8).

**Definition of Done:**
- [ ] `src/parsers/exr.ts` возвращает `Exercise { title, video, audio, sequences }`.
- [ ] `src/sequence-runner.ts` исполняет минимальный набор опкодов на тестовом примере.
- [ ] Список 16 упражнений рендерится, каждое имеет кликабельную строку.
- [ ] Клик на упражнение → правильное видео + аудио воспроизводятся.
- [ ] Возврат «← к песне» сохраняет позицию в песне.
- [ ] Difficulty rects рендерятся поверх tab strip с правильной геометрией.
- [ ] Клик на rect → открывает связанное упражнение (`exercice=N` → `HeyJoeEx<N>`).

**Acceptance Criteria:**
- [ ] В песне вижу 16 пунктов на боковой панели + цветные подсветки на табах.
- [ ] Клик на упражнение 3 → новый экран, играет видео урока, слышу голосовой комментарий.
- [ ] Жму «следующее» → переключаюсь на упражнение 4 без перезагрузки страницы.
- [ ] Возврат → песня снова на той позиции, где я её оставил.
- [ ] Hover на жёлтую подсветку в табах → tooltip «Упражнение 4: <название>». Клик → переход в это упражнение.
- [ ] Slow-down работает и в режиме упражнения.

**Риски / нюансы:**
- Sequence-runner может оказаться сложнее, чем кажется. Если в EXR Hey Joe есть нестандартные опкоды — упростить (статичный плеер вместо runner). Решение принимаем во время этапа.
- Картинки табулатуры в EXR (`sco1-2.bmp`) — отдельные BMP, нужны в `public/assets/`.

## Phase 5 — Тюнер

**Цель:** настроить гитару через микрофон без выхода из приложения.

**Включено:**
- Запрос разрешения микрофона (`getUserMedia`).
- Pitch detection: **YIN-алгоритм** (FFT для гитарных низких струн неточен; YIN — стандарт для тюнеров).
- UI:
  - 6 кнопок струн (E2 82.4, A 110, D 146.8, G 196, B 246.9, E4 329.6).
  - Большая стрелка / индикатор текущей частоты.
  - Цвет: зелёный если в пределах ±5 cent, жёлтый ±20, красный за пределами.
  - Текущая нота + точное отклонение в центах.
  - Кнопка «🔊» возле каждой струны → проигрывает эталон из CHD-сэмплов.
- Без записи звука куда-либо (privacy-first).

**Исключено:**
- Альтернативные строи (Drop D, Open G и т.д.).
- Octave detection для 12-струнок.

**Definition of Done:**
- [ ] `src/audio/pitch-yin.ts` имплементирует YIN, юнит-тест на синусоиде 440 Гц → возвращает 440 ±0.5 Гц.
- [ ] Получение микрофона работает (HTTPS или localhost требование).
- [ ] Стабильное определение струны при чистом дёргании (E2..E4) с погрешностью ≤2 cent.
- [ ] UI обновляется ~30 раз в секунду.

**Acceptance Criteria:**
- [ ] Открываю тюнер → запрос разрешения микрофона.
- [ ] Дёргаю E (низкую) → видно «E 82.4 Hz», стрелка по центру если настроено.
- [ ] Расстраиваю → стрелка уходит, цвет краснеет, цифра в центах меняется (например +15 ¢).
- [ ] Жму 🔊 рядом с A → слышу эталон 110 Гц.
- [ ] Тюнер не падает при тишине (мьют → стрелка серая или прячется).

**Риски / нюансы:**
- Octave error на низком E2 — самый частый баг тюнеров. YIN это решает, но требует калибровки порога.
- На MacBook микрофоны разные; протестировать минимум на встроенном + внешнем.

## Phase 6 — Метроном

**Цель:** независимый метроном для упражнений.

**Включено:**
- Слайдер + цифровой ввод BPM (диапазон 30–240).
- Кнопки «+1», «-1», «+5», «-5».
- Старт/стоп.
- Тик через `AudioContext.currentTime + scheduleNext()` (точность ±1мс).
- Визуальный flash на каждом тике.
- Sample тика — оригинальный `TICK.WAV` из `METRONOM/`.
- Опционально: разные тики для downbeat (1-я доля) и offbeat (2/3/4) — позже.

**Исключено:**
- Размер не 4/4 (можно потом).
- Subdivisions (восьмые/триоли) — можно потом.

**Definition of Done:**
- [ ] `src/audio/metronome.ts` использует look-ahead scheduler (planLookahead=100ms, scheduleAheadTime=25ms).
- [ ] Старт/стоп без артефактов и фантомных тиков.
- [ ] Изменение BPM применяется со следующего тика.
- [ ] Визуальный flash синхронен (использовать тот же `AudioContext.currentTime`).

**Acceptance Criteria:**
- [ ] Ставлю 80 BPM, жму Start → слышу ровные тики каждые 750мс.
- [ ] Жму +5 → стало 85 BPM, тики стали чаще.
- [ ] Жму Stop → тики прекращаются мгновенно.
- [ ] Визуальный flash совпадает с щелчком в наушниках.

**Риски / нюансы:**
- На Safari `AudioContext` создаётся в suspended — нужен `resume()` после первого user gesture.

## Workflow выполнения

```
[Opus]   → пишет phases/N-spec.md (детальная спека для одного этапа)
                     ↓
[Opus]   → Agent({ model: "sonnet", prompt: "Read phases/N-spec.md and implement" })
                     ↓
[Sonnet] → читает спеку, пишет код, отчитывается
                     ↓
[Opus]   → проверяет:
              • DoD-чеклист (читает изменённые файлы)
              • AC прогоняет вручную/просит пользователя смок-тест
              • если ок → следующий этап
              • если нет → конкретные правки в новый prompt → тот же агент дорабатывает
                     ↓
[User]   → реально пользуется приложением, даёт фидбек
```
